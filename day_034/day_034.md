## 3. Market the hardware product
Product: `EV Battery Management System (EV BMS)`

Post idea:
An advanced BMS supports daisy-chained cell monitoring to reduce wiring and connector complexity.

Each monitoring IC sits on a cell group and communicates serially to the main controller.

This architecture scales cleanly from small modules to large battery packs.

## 4. Circuit scenario + SPICE
Scenario:
A daisy-chainable battery monitor IC configuration with isoSPI communication.

Post idea:
High-voltage battery packs require multiple monitoring ICs stacked in series. The isoSPI interface allows daisy-chain communication over a single twisted pair.

SPICE can be used to study the signal integrity, noise margin, and propagation delay along the daisy chain as the number of ICs increases.

This topology reduces isolation cost because only the bottom IC needs galvanic isolation to the main controller.

## 6. Analog IC datasheet analysis
Suggested IC: `LTC6804`

Technical idea:
The `LTC6804` is a 12-channel battery monitor IC that can be evaluated for cell voltage measurement, temperature sensing, and daisy-chain communication in EV BMS designs.

Key datasheet parameters to analyze:
- Voltage measurement accuracy and resolution
- Measurement conversion time and noise rejection
- Maximum input voltage and input protection
- isoSPI communication data rate and cable length
- Temperature sensor interface
- Quiescent current and sleep mode power

In an EV BMS, the LTC6804 provides a complete analog front-end solution that simplifies cell monitoring while maintaining high accuracy across temperature and voltage.
